How to setup Jenkins on Kubernetes

Kapila Nishantha

Aug 20, 2019·2 min read

Jenkins is a one of leading open source automation server. Kuberenetes is open source platform which enable automating deployments, scaling and management of the containers.

This guid will walk you through step by step process of deploying Jenkins on Kubernetes cluster.

Create depolyment.yaml

deployement.yaml contain deployment details about jenkins.

Kind , define config file type, since this is deployement.yaml, Kind value should be deployment.Then add docker image details under container section. Here i’m using jenkins/jenkins:lts image and expose port 8080 and 5000.

Creating a persistance volume is must have , because all the jobs , library and other jenkins data should be sync with other nodes and data should be persist. In this example i already created NFS server and pointing that location as a persistance volume. AccessModes should be ReadWriteOnce, otherwise jenkins cannot access volumes.

Volume claim is a request for storage by a user.Claims can request specific size and access modes. Allocate 1Gi by setting storage under resources section.

Service is a abstract way to expose an application running on a pods as a network service.So we need to define service for jenkins then third parties can invoke using this service.Here exposing port 8080 and 5000. type is ClusterIp.

Creating ingrees is for expose jenkins to public. Ingress expose HTTP and HTTPS routes from the outside the cluster to service within the cluster.

After adding above configuration properly, jenkins management console should be access on your browser.

After carefly reading above configurations and steps, you should be able to understand way of deploying docker container to kube cluster not only jenkins.

Deploying docker container, attaching persistence volume, attaching service and attaching ingress. those steps will be simple scenario i explained using jenkins use case.
